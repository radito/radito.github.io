Apa itu Rooting ?#

Rooting pada perangkat atau sistem terbenam adalah proses untuk mendapatkan akses ke sistem operasi perangkat yang memungkinkan pengguna untuk memodifikasi file sistem dan pengaturan yang sebelumnya dibatasi. Rooting dapat memberikan manfaat seperti menginstal firmware kustom, menghapus perangkat lunak yang telah diinstal sebelumnya, menyesuaikan antarmuka pengguna perangkat, dan menyesuaikan pengaturan sistem.

Langkah - Langkah Akses Root Bolt BL-100#

üí° Versi Firmware yang digunakan adalah HW: rev 2 | SW: BOLT!4G v3.1.32

  1. Akses Web GUI router dengan mengetikkan IP Adress router ke browser

  1. Pergi ke menu Settings ‚Üí Administration

  1. Pergi ke menu Configuration ‚Üí Lalu terdapat button Export ‚Üí Klik button Export

  1. Setelah filter terdownload lalu buka file tar tersebut dengan code editor. Disini saya menggunakan Visual Studio Code, anda dapat menggunakan Vscode ataupun code editor lain, yang penting support file binary.

  2. Setelah filter terbuka klik CTRL + F lalu cari keyword LAN_Telnet

  1. Ubah nilainya menjadi 1 sehingga LAN_Telnet=1

  1. Kemudian cari lagi dengan keyword Login=

  1. Nah pada line Login dan Password tersebut merupakan akun untuk login ke root nya, catat nilainya untuk dipakai nanti, disini contohnya. Setiap router bisa berbeda jadi pastikan lagi di file konfigurasi anda.
Login=admin
Password=db40

  1. Jika sudah klik simpan pada file yang telah diubah.

  2. Akses kembali halaman Configuration di Website router. Dan import file yang telah dimodifikasi tadi.

  1. Klik Upload dan router akan memproses file konfigurasi, setelah selesai maka router akan reboot dengan sendirinya dengan konfigurasi yang telah dimodifikasi.

Testing Login ke Root#

Cek scan port pada router, muncul port 23 dimana port telnet router tersebut dijalankan. Lalu kita coba test login via telnet.

Yes, berhasil login ke router dengan user yang didapatkan dari file konfigurasi tadi (admin/db40). Kita coba check apakah kita mendapatkan privilege root dengan command id. Dan kita tes beberapa command lain.

Ternyata user admin memiliki uid=0 dan gid=0 artinya kita memiliki akses dengan level root, dan di file passwd hanya ada satu user saja yaitu admin.
